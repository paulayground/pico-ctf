# 배운 점

- 스테가노그래피: 사진, 음악, 동영상 등의 파일 안에 데이터를 숨기는 기술

  동영상, 사진, 오디오 같은 파일에는 불필요한 데이터들도 상당수 포함되어 있어, 이런 부분들을 활용해 다른 메시지로 대치해 겉으로 잘 안보이게한다.

- steghide: 스테가노그래피 기술을 적용하거나 해독하는 툴

# 풀이 과정 기록

1. `exiftool`을 활용해 메타데이터에 대한 내용을 먼저 확인하고 `comment`부분의 `base64` decoding까지는 무리없이 진행이 되었다.

2. 하지만 그 이후로는 별 다른 정보를 얻을 수 없어 파일의 내용을 확인을 해봐야하나 확인이 들었고 16진수나 특이한 문자열을 확인해보려 찾아봤지만, 어떤 것을 찾아야하는 지, 어떤게 특이한 점인지 찾을 수가 없어 일단 디코딩하면서 의심되는 단어인 `steghide`를 찾다가 스테가노그래피에 대한 힌트를 얻었다.

3. 스테가노그래피 적용 기술을 해독하는 방법에 대한 내용을 검색하다 `steghide`를 통해 해독이 가능하다는 것을 알았고 flag를 찾을 수 있었다.

# 심화 학습 (Deep Dive)

원본과 복사본이 존재한다면 체크섬 값을 확인하여 변조여부를 확인할 수 있을 것 같다.

??? 복사본이 없는 경우 이미지를 보고 스테가노 그래피가 적용되어 있는지 알 수 있지?

!!! jpeg 파일 시그니처 FF D8 FF E0 / FF D9 를 통해 정상적인 파일 시그니쳐를 가지는 지 확인할 수 있고, 푸터 시그니쳐 뒤에 데이터를 은닉하는 경우도 있다.

`binwalk` 같은 툴을 사용 하여 시그니쳐, 엔트로피 확인도 가능하다

일반적인 이미지파일은 인접한 픽셀끼리 색상이 비슷해서 어느정도 규칙성(낮은 엔트로피)가 있는데, 숨겨진 데이터의 경우 무작위적인 데이터들이 들어가 낮았던 엔트로피가 갑자기 높아져 이상함을 알 수 있다.

```bash
# img.jpg 시그니쳐 확인
# 정상적인 JPEG헤더로 이 외의 추가적인 파일 시그니처가 없어 정상이라는 것을 알 수 있다.
$ binwalk img.jpg 

DECIMAL       HEXADECIMAL     DESCRIPTION
--------------------------------------------------------------------------------
0             0x0             JPEG image data, JFIF standard 1.01

# img.jpg 엔트로피
# 0x400부근에서 엔드로피 0.966으로 1에 가깝게 올라 의심할 수 있는 부분이 있다고 생각할 수 있다.
$ binwalk -E img.jpg 
DECIMAL       HEXADECIMAL     ENTROPY
--------------------------------------------------------------------------------
1024          0x400           Rising entropy edge (0.966249)
```
