# 배운 점

임의적으로 작성된 문자들을 볼 때 규칙적으로 반복되는 경우가 있는 `base64`나 `hex`값의 경우 인코딩되어있나 의심해보는 관점을 가져야겠다.

# 풀이 과정 기록

`logs.txt`를 `binwalk`로 시그니처 찾아보고 또 시그니처들이 여러개 있고 중간에 특이한 시그니쳐들이 들어있는 것을 보고 해당 구간을 꺼내보기도 하고 꺼내서 다시 확인하고를 반복했지만 단서를 찾을 수 없었다.
막혀있는 도중에 `logs.txt`를 `base64` 디코딩하라는 힌트를 보고 진행이 되었다.
디코딩된 결과의 파일 타입을 확인하니까 `png`였고 열어본 결과 이미지 안에 문자값이 있었으며, 파일 내용에 문자열에 해당되는 부분이 있나 찾아보려고 시도하다 우연하게 string으로 변환되어 이게 hex된 값인걸 알게되었고, 허무하게 flag를 획득했다.
